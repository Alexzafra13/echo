# Material de defensa del TFG — Echo

Carpeta con el material de apoyo para la exposición oral del proyecto.

## Contenido

| Archivo                                          | Qué es                                                                 |
| ------------------------------------------------ | ---------------------------------------------------------------------- |
| [`documento-defensa.md`](./documento-defensa.md) | Guion técnico completo: arquitectura, decisiones, código comentado y **anexo de preguntas del tribunal con respuestas**. |
| [`presentacion.html`](./presentacion.html)       | Presentación de diapositivas (reveal.js), lista para proyectar.        |

## Cómo usar la presentación

Es un archivo HTML autocontenido (carga reveal.js por CDN, así que necesita
conexión a internet la primera vez).

1. **Abrir**: doble clic en `presentacion.html` (se abre en el navegador), o
   ábrelo desde el navegador con `Archivo → Abrir`.
2. **Navegar**: flechas `←` / `→` (o `Espacio` para avanzar).
3. **Notas del orador**: pulsa la tecla `S` para abrir la vista de presentador
   (verás tus notas y un cronómetro en una ventana aparte).
4. **Pantalla completa**: pulsa `F`.
5. **Vista general** de todas las slides: pulsa `Esc` o `O`.
6. **Imprimir a PDF**: abre `presentacion.html?print-pdf` en Chrome y usa
   `Imprimir → Guardar como PDF` (por si necesitas una copia estática).

> Cada diapositiva incluye **notas del orador** (lo que decir) accesibles con `S`.

## Consejos rápidos para la defensa

- Prepara una **demo en vivo** (o vídeo de respaldo): login → navegar biblioteca →
  reproducir → Wave Mix → progreso de escaneo.
- Ten abierto el **Swagger** (`/api/docs`) por si preguntan por la API.
- Domina tres frases: el **pitch**, qué es la **arquitectura hexagonal** y por qué
  el trabajo pesado es **asíncrono**.
- En el documento tienes el **guion cronometrado** (Anexo C) y las **preguntas del
  tribunal** (Anexo B).
